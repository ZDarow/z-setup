# ✅ Чеклист анализа APK

## 📋 Предварительная подготовка

- [ ] Получить APK файл легально
- [ ] Создать рабочую директорию для анализа
- [ ] Проверить наличие необходимых инструментов
- [ ] Убедиться в наличии свободного места (5-10 ГБ)

## 🔧 Проверка инструментов

### Обязательные инструменты
- [ ] `apktool` — декомпиляция ресурсов
- [ ] `jadx` — декомпиляция в Java
- [ ] `unzip` — извлечение файлов
- [ ] `strings` — анализ строк
- [ ] `grep` — поиск паттернов
- [ ] Java JDK 17+

### Опциональные инструменты
- [ ] `frida` — динамический анализ
- [ ] `jadx-gui` — графический интерфейс
- [ ] `bytecode-viewer` — просмотр байт-кода
- [ ] `apktool-m` — модифицированный apktool

## 📦 Этап 1: Первичный анализ

- [ ] Проверить размер APK файла
- [ ] Извлечь список файлов из APK
- [ ] Получить информацию через `aapt dump badging`
- [ ] Извлечь AndroidManifest.xml
- [ ] Определить package name
- [ ] Определить версии (code, name)
- [ ] Определить min/target SDK

## 🔓 Этап 2: Декомпиляция

- [ ] Декомпилировать через `apktool d`
- [ ] Декомпилировать в Java через `jadx -d`
- [ ] Проверить успешность декомпиляции
- [ ] Убедиться в наличии AndroidManifest.xml
- [ ] Убедиться в наличии Java файлов

## 📋 Этап 3: Анализ разрешений

### Bluetooth-разрешения
- [ ] Проверить `BLUETOOTH`
- [ ] Проверить `BLUETOOTH_ADMIN`
- [ ] Проверить `BLUETOOTH_SCAN` (Android 12+)
- [ ] Проверить `BLUETOOTH_CONNECT` (Android 12+)
- [ ] Проверить `BLUETOOTH_ADVERTISE` (Android 12+)
- [ ] Проверить `ACCESS_FINE_LOCATION`
- [ ] Проверить `ACCESS_COARSE_LOCATION`

### Другие важные разрешения
- [ ] Проверить разрешения на SMS
- [ ] Проверить разрешения на контакты
- [ ] Проверить разрешения на хранилище
- [ ] Проверить разрешения на телефон
- [ ] Проверить разрешения на камеру
- [ ] Проверить разрешения на микрофон

## 🔍 Этап 4: Поиск Bluetooth-кода

### Classic Bluetooth
- [ ] Найти `BluetoothAdapter`
- [ ] Найти `BluetoothDevice`
- [ ] Найти `BluetoothSocket`
- [ ] Найти `BluetoothServerSocket`
- [ ] Найти `createRfcommSocket`
- [ ] Найти `createInsecureRfcommSocket`

### Bluetooth Low Energy
- [ ] Найти `BluetoothLeScanner`
- [ ] Найти `ScanCallback`
- [ ] Найти `ScanFilter`
- [ ] Найти `BluetoothGatt`
- [ ] Найти `BluetoothGattCallback`
- [ ] Найти `connectGatt`
- [ ] Найти `BluetoothGattCharacteristic`
- [ ] Найти `BluetoothGattService`
- [ ] Найти `readCharacteristic`
- [ ] Найти `writeCharacteristic`
- [ ] Найти `setCharacteristicNotification`

### UUID
- [ ] Найти UUID в формате XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX
- [ ] Найти `UUID.fromString()` вызовы
- [ ] Найти 16-битные UUID (0xXXXX)
- [ ] Классифицировать UUID (сервисы/характеристики/дескрипторы)

## 📁 Этап 5: Анализ компонентов

- [ ] Изучить все Activity
- [ ] Изучить все Services (особенно Bluetooth-сервисы)
- [ ] Изучить все Receivers
- [ ] Изучить все Providers
- [ ] Найти Intent Filters для Bluetooth действий

## 🔎 Этап 6: Анализ строк

- [ ] Извлечь все строки из APK
- [ ] Найти URL и API endpoints
- [ ] Найти IP-адреса
- [ ] Найти hardcoded credentials
- [ ] Найти Bluetooth-related строки
- [ ] Найти команды/протоколы обмена

## 🔐 Этап 7: Безопасность

- [ ] Проверить на hardcoded ключи
- [ ] Проверить на незашифрованные данные
- [ ] Проверить SSL/TLS конфигурацию
- [ ] Проверить проверку сертификатов
- [ ] Проверить obfuscation (ProGuard/R8)

## 🧪 Этап 8: Динамический анализ (опционально)

- [ ] Установить APK на устройство/эмулятор
- [ ] Настроить Frida server
- [ ] Написать скрипт для перехвата Bluetooth-вызовов
- [ ] Запустить мониторинг Bluetooth-активности
- [ ] Записать логи обмена данными

## 📊 Этап 9: Документирование

- [ ] Заполнить шаблон отчёта
- [ ] Сохранить найденные UUID
- [ ] Сохранить список критических файлов
- [ ] Сохранить примеры кода
- [ ] Создать диаграмму последовательности
- [ ] Сфотографировать UI (если доступно)

## 📝 Этап 10: Выводы

- [ ] Определить тип Bluetooth (Classic/BLE/Dual)
- [ ] Описать протокол обмена данными
- [ ] Выявить уязвимости
- [ ] Сформулировать рекомендации
