## What I do
- Создаю или обновляю README.md на основе кода
- Генерирую JSDoc/докстринги для функций и классов
- Пишу понятные комментарии к сложной логике
- Составляю примеры использования
- Обновляю документацию при изменении API/интерфейсов

## When to use me
Используй, когда нужно задокументировать новый или существующий код, создать README или API-документацию.

## Как я работаю
1. **Анализ кода** — читаю структуру проекта, API-эндпоинты, сигнатуры функций
2. **Сбор контекста** — определяю назначение модулей, поток данных, зависимости
3. **Генерация** — создаю README с разделами: описание, установка, использование, API, примеры
4. **Валидация** — проверяю: все ли публичные функции задокументированы, нет ли устаревшей информации

## Формат README
```markdown
# Название проекта
Краткое описание (1-2 предложения)

## Установка
```bash
команда установки
```

## Использование
Базовые примеры

## API (если применимо)
Сигнатуры функций/эндпоинтов с описанием

## Примеры
Код с комментариями
```
